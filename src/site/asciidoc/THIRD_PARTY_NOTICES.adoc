= IKE Tooling — Third-Party Notices
:icons: font
:source-highlighter: coderay
:toc: preamble
:toclevels: 3

This document acknowledges third-party open-source software that
`ike-tooling` depends on, links against, or relies on at build
time. The Maven `licenses.html` and `dependencies.html` reports
generated by `mvn site` cover declared `<dependencies>` mechanically;
this document covers the rest — Maven core machinery, plugins, the
site skin, build-time signing — that the reports either don't reach
or under-report.

For the corresponding notices in the rest of the IKE platform see:

* `link:../ike-docs/THIRD_PARTY_NOTICES.html[ike-docs]` — AsciiDoc
  rendering chain, fonts, DocBook, frontend assets.
* `link:../ike-platform/THIRD_PARTY_NOTICES.html[ike-platform]` —
  Java toolchain, BOM-managed dependencies, test framework.

== Maven Site skin

|===
| Component | License | Role

| https://github.com/sentrysoftware/maven-skins[Sentry Maven Skin]
| Apache 2.0
| The Maven Site skin applied to every published page. Provides the
  base navigation chrome (header, sidebar, breadcrumbs). The IKE
  Forest theme overlays Sentry's defaults via the `site-theme`
  classifier produced by `ike-build-standards` (see #318).

| https://github.com/sentrysoftware/maven-skins/tree/main/maven-skin-tools[Maven Skin Tools]
| Apache 2.0
| Velocity helper context (`$headElement`, `$bodyElement`) used by
  the Sentry skin. Pulled in as a `maven-site-plugin` dependency.

| https://www.graalvm.org/[GraalVM Polyglot, GraalVM JS]
| GPL 2.0 with Classpath Exception (Community Edition)
| Replaces the deprecated Nashorn engine for skin script evaluation.
  Pulled in as a `maven-site-plugin` dependency.

| https://prismjs.com/[Prism]
| MIT
| Syntax highlighter shipped in rendered HTML. Used by the AsciiDoc
  pipeline; per-language CSS is bundled in the rendered output.
|===

== Cryptographic signing

|===
| Component | License | Role

| https://www.bouncycastle.org/[Bouncy Castle (`bcprov-jdk18on`, `bcpg-jdk18on`)]
| MIT-style (Bouncy Castle license)
| OpenPGP signing engine used by `maven-gpg-plugin`'s `bc` signer.
  IKE releases consistently use `bc` because the native `gpg` signer
  serializes through `gpg-agent` and breaks parallel reactor builds.
|===

== Documentation tooling consumed during plugin development

The ike-tooling reactor itself does not render AsciiDoc to PDF or
HTML — that work belongs to `ike-docs`. The Maven Site of this
reactor consumes:

|===
| Component | License | Role

| https://github.com/asciidoctor/asciidoctor-doxia[Asciidoctor Parser Doxia Module]
| MIT
| Lets `maven-site-plugin` render the `src/site/asciidoc/*.adoc`
  files (this page among them) as part of `site:site`.

| https://www.jruby.org/[JRuby]
| EPL 2.0 / GPL 2.0 / LGPL 2.1 (multi-licensed)
| Embedded Ruby runtime that the AsciiDoc parser uses. Pulled in
  transitively by the Doxia module.
|===

== Verification

* `mvn site` populates `target/site/licenses.html` from declared
  `<dependencies>`. The Apache- and MIT-licensed entries above
  appear there with full text.
* `mvn site` populates `target/site/plugins.html` and
  `target/site/plugin-management.html` from declared
  `<build><plugins>` and `<build><pluginManagement>`.
* This curated document complements those mechanical reports.
  Issues or omissions: file an issue at
  https://github.com/IKE-Network/ike-issues.
